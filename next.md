# CypherGuard AI - 下一步开发计划

## 📊 当前系统状态总结

### ✅ 已完成功能

#### 前端 (Frontend)
- ✅ 用户认证系统（注册/登录，支持邮箱或用户名登录）
- ✅ 知识库管理页面（文档上传、列表、删除）
- ✅ AI 对话页面（创建对话、发送消息、查看引用）
- ✅ 响应式 UI 设计（Tailwind CSS + Framer Motion）
- ✅ API 服务层（Axios + 拦截器）
- ✅ 路由保护（ProtectedRoute）
- ✅ 认证上下文（AuthContext）

#### 后端 (Backend)
- ✅ 认证服务（Auth Service）- JWT Token、用户注册/登录
- ✅ 知识库服务（KB Service）- 文档上传、MinIO 存储、任务管理
- ✅ 对话服务（Chat Service）- RAG 检索、Rerank、LLM 生成
- ✅ 模型网关（Model Gateway）- 统一模型访问、限流、并发控制
- ✅ 异步任务（Worker）- 文档解析、切分、向量化、索引
- ✅ 数据库模型（PostgreSQL）- 用户、文档、对话、消息等
- ✅ CORS 配置（所有服务）
- ✅ Nginx 网关（路由、负载均衡）

#### 基础设施 (Infrastructure)
- ✅ Docker Compose 编排
- ✅ PostgreSQL 数据库
- ✅ Redis 缓存和任务队列
- ✅ MinIO 对象存储
- ✅ Qdrant 向量数据库
- ✅ Celery 异步任务

## ⚠️ 当前系统不成熟的地方

### 1. 🤖 模型集成（最关键）

**问题：**
- Model Gateway 使用的是 **Mock 数据**，没有真实的 AI 模型
- Embedding 生成是基于文本哈希的伪向量
- Chat 生成返回固定的 Mock 响应
- Rerank 使用简单的词频匹配

**影响：**
- ❌ AI 对话无法给出真实的智能回答
- ❌ 向量检索不准确
- ❌ 文档相关性排序不准确
- ❌ 系统核心功能无法真正使用

**需要：**
- 集成真实的 Embedding 模型（如 OpenAI、Cohere、本地模型）
- 集成真实的 LLM（如 GPT-4、Claude、本地 LLM）
- 集成真实的 Rerank 模型（如 Cohere Rerank、本地模型）

### 2. 🔧 模型配置管理

**问题：**
- 没有模型配置管理界面
- 无法动态添加/切换模型
- API Key 管理不完善
- 模型参数无法调整

**需要：**
- 模型配置管理页面（前端）
- 模型配置 CRUD API（后端）
- API Key 加密存储
- 模型测试功能

### 3. 📝 文档处理能力有限

**问题：**
- 仅支持基础文本提取
- PDF 解析可能不完整（表格、图片）
- 没有 OCR 支持
- 没有文档预览功能
- 切分策略简单（固定大小）

**需要：**
- 改进 PDF 解析（支持表格、图片）
- 添加 OCR 支持
- 智能切分策略（按段落、章节）
- 文档预览功能

### 4. 🔍 检索质量

**问题：**
- 没有混合检索（仅向量检索）
- 没有查询改写
- 没有多轮对话上下文
- 没有检索结果缓存

**需要：**
- 混合检索（向量 + 关键词）
- 查询改写和扩展
- 对话上下文管理
- 检索结果缓存

### 5. 👤 用户体验

**问题：**
- 没有文档预览
- 没有对话历史搜索
- 没有引用高亮
- 没有导出功能
- 没有系统设置页面

**需要：**
- 文档预览（PDF、TXT）
- 对话历史搜索和过滤
- 引用内容高亮
- 对话导出（Markdown、PDF）
- 系统设置页面

### 6. 🔐 安全和权限

**问题：**
- 没有细粒度权限控制
- 没有文档共享功能
- 没有审计日志
- API Key 明文存储

**需要：**
- RBAC 权限系统
- 文档共享和协作
- 操作审计日志
- API Key 加密存储

### 7. 📊 监控和日志

**问题：**
- 没有系统监控
- 没有性能指标
- 日志分散
- 没有错误追踪

**需要：**
- 系统监控（Prometheus + Grafana）
- 性能指标收集
- 集中式日志（ELK）
- 错误追踪（Sentry）

### 8. 🧪 测试覆盖

**问题：**
- 没有单元测试
- 没有集成测试
- 没有 E2E 测试
- 没有性能测试

**需要：**
- 单元测试（pytest）
- 集成测试
- E2E 测试（Playwright）
- 性能测试（Locust）
